# TrueUsed Roadmap

TrueUsed 当前阶段已经跑通二手交易主链路，下一步目标不是继续堆页面，而是把演示型项目收口成可稳定运行、可验证、可交付的工程基线。

## 阶段定位

| 阶段 | 目标 | 状态 |
| --- | --- | --- |
| P0 基线收口 | 文档、运行口径、Java 版本、交付检查统一 | 进行中 |
| P1 安全与交易一致性 | 权限边界、支付幂等、下单并发保护 | 待启动 |
| P2 性能与真实数据闭环 | 缓存一致性、业务监控、前端占位数据清理 | 待启动 |
| P3 AI/RAG 增强 | 智能客服、商品发布辅助、定价建议 | 待启动 |

## 模块现状

### 后端基础设施

已完成：

- Spring Boot 3.5 后端服务。
- MySQL、Redis、Flyway 基础接入。
- Actuator、Prometheus、Grafana 基础监控。
- Docker Compose 一键启动后端、前端、数据库、缓存和监控组件。

风险：

- 本地运行依赖 Java 21，但部分旧文档仍可能残留 Java 17 表述。
- 生产配置与开发配置尚未完全拆分。
- 当前默认环境变量仍保留演示值，不适合生产环境直接使用。

下一步：

- 将 Java 版本、Docker 镜像和 README 统一到 Java 21。
- 拆分 `dev` / `prod` profile，生产缺少关键密钥时 fail fast。
- 为发布前检查增加固定 release checklist。

验收方式：

- `./mvnw test` 通过。
- `docker compose up -d --build` 后后端健康检查为 `UP`。
- `docs/release-checklist.md` 中 P0 项全部勾选。

### 用户与认证

已完成：

- 用户注册、登录、个人信息基础能力。
- JWT access token 与 refresh token。
- refresh token 轮换与撤销表持久化。
- 登出撤销当前 token。
- WebSocket 握手鉴权。

风险：

- 路由级权限与 Controller 内手动鉴权混用。
- 部分公开接口范围偏宽，需要按 HTTP 方法细化。

下一步：

- 收紧 `SecurityConfig` 中商品、用户、订单等接口的权限边界。
- 增加 Controller 权限测试，覆盖匿名、普通用户、管理员三类身份。

验收方式：

- 匿名用户只能访问公开 GET 接口。
- 非资源归属用户访问订单、地址、商品编辑接口返回 403。
- 管理端接口只有 `ROLE_ADMIN` 可访问。

### 商品与交易

已完成：

- 商品发布、上下架、详情、搜索、收藏。
- 卖家自出与平台验货两种交易模式。
- 下单、支付、发货、确认收货基础闭环。
- 商品静态信息与动态计数 Redis 缓存。

风险：

- 高并发下同一商品可能出现竞争下单风险。
- Redis 浏览量、收藏数与数据库回写策略需要明确。

下一步：

- 下单时增加数据库级状态更新保护或悲观锁。
- 增加商品缓存降级与回写策略。
- 针对同一商品并发下单补集成测试。

验收方式：

- 20 个买家并发购买同一商品，仅 1 个订单创建成功。
- Redis 不可用时商品详情仍可从数据库返回。
- 商品删除、下架、改价后缓存不会返回旧状态。

### 支付、钱包与售后

已完成：

- 钱包充值、支付密码、钱包支付。
- 支付宝沙箱入口。
- 支付策略拆分。
- 退款申请、卖家审批、拒绝后重提、mock 完成退款。

风险：

- 第三方支付回调的金额、订单号、重复通知校验仍需强化。
- 逆向物流仍是 mock，不应对外宣称生产能力。

下一步：

- 补支付宝回调幂等与金额校验测试。
- 明确 `REFUND_ONLY` 与 `RETURN_REFUND` 的状态流转边界。
- 将 mock 退款行为在文档和前端文案中保持一致。

验收方式：

- 重复支付回调不会重复入账或重复推进订单。
- 非订单买家无法支付订单。
- 卖家只能处理属于自己的售后申请。

### 验货、物流与消息

已完成：

- 平台验货报告与订单绑定。
- mock 物流节点生成与展示。
- 通知、消息和客服中心基础页面。

风险：

- 验货与物流仍偏 mock，需要避免与真实第三方能力混淆。
- 消息系统还不是完整工单系统。

下一步：

- 明确验货报告字段来源和 mock 边界。
- 为物流状态流转补充测试。
- 统一客服消息与帮助中心知识口径。

验收方式：

- 平台验货订单详情能稳定展示验货报告、物流轨迹和售后入口。
- 未到可签收节点时确认收货失败。
- 帮助中心说明与后端真实状态流转一致。

### 前端

已完成：

- Vue 3 + Vite 桌面端前端。
- 首页、商品详情、结算、订单、钱包、验货报告、售后、帮助中心等主页面。
- Axios 统一请求封装，兼容后端标准响应。

风险：

- 部分页面仍有 mock 或“待接真实数据”文案。
- 缺少系统化 E2E 与视觉回归测试。

下一步：

- 优先清理个人中心、商品详情、浏览历史、退款详情中的占位数据。
- 增加核心交易流 E2E。
- 保持 UI 风格向专业、高信息密度 Dashboard/IDE 风格收束。

验收方式：

- 主交易链路无明显 mock 统计误导。
- `npm run build` 通过。
- 买家下单、钱包支付、卖家发货、确认收货、退款申请核心路径可跑通。

### AI/RAG 增强

已完成：

- 暂未接入。

风险：

- 在交易链路未加固前接 AI，容易把问题复杂化。
- LLM 输出需要受规则和订单上下文约束，不能直接替代交易决策。

下一步：

- 先做低风险 FAQ RAG：帮助中心、售后规则、订单状态流转。
- 再做商品标题/描述辅助生成。
- 最后考虑定价建议和风控解释。

验收方式：

- AI 回答必须返回引用来源。
- 不能编造订单状态。
- 不能直接执行退款、改价、发货等高风险动作。
